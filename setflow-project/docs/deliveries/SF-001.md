# SF-001 — Inicializar projeto Android

## Estado
DONE

## Contexto
O projeto possui documentação fundacional, seed inicial e ADRs aceitos, mas ainda não possui projeto Android executável.

## Objetivo
Criar um projeto Android nativo mínimo que abra no Android Studio, sincronize com Gradle e compile um app Compose inicial.

## Referências obrigatórias
- FR-001
- NFR-001
- NFR-010
- NFR-011
- ADR-001
- ADR-002
- ADR-003
- `docs/02-mvp-scope.md`
- `docs/07-technical-architecture.md`

## Pré-condições
- Android Studio instalado.
- Android SDK com plataforma 36 e Build Tools 36.0.0.
- JDK disponível pelo Android Studio ou configurado no ambiente.

## Escopo obrigatório
- Criar projeto Gradle Kotlin DSL com módulo único `app`.
- Configurar Android application com `minSdk 26`, `compileSdk 36` e `targetSdk 36`.
- Configurar version catalog.
- Criar tela Compose mínima.
- Definir app label e package inicial como Repcore: `com.douglas.repcore`.
- Não declarar permissão `INTERNET`.

## Fora do escopo
- Room, DataStore, Hilt, Navigation e importação do seed.
- Telas reais de treino, histórico ou configurações.
- Assinatura release.
- Migração completa dos documentos de SetFlow para Repcore.

## Regras de implementação
- Usar Kotlin/Compose e Material 3.
- Manter textos em recursos.
- Não criar backend, analytics ou permissões remotas.
- Não criar multi-module neste momento.

## Arquivos ou áreas esperadas
- `settings.gradle.kts`
- `build.gradle.kts`
- `gradle/libs.versions.toml`
- `app/`

## Critérios de aceite
1. O projeto possui módulo Android `app`.
2. O app possui `MainActivity` em Compose.
3. `minSdk`, `compileSdk` e `targetSdk` seguem a arquitetura.
4. Não existe permissão de internet no manifest.
5. O projeto executa `assembleDebug` em ambiente Android configurado.

## Testes obrigatórios
- Build debug.
- Inspeção manual do manifest para permissões.

## Comandos de validação
```bash
./gradlew assembleDebug
```

## Definition of Done específica
- Android Studio consegue abrir e sincronizar o projeto.
- APK debug pode ser instalado em aparelho conectado.

## Resultado
- Projeto Android inicial criado na raiz do repositório.
- App label e package definidos como Repcore.
- `assembleDebug` executado com sucesso.
- `testDebugUnitTest` executado com sucesso, sem testes ainda.
